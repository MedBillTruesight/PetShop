# Developer Experience (DX) Documentation

Welcome to the Pet Shop API Developer Experience documentation. This folder contains comprehensive, public-facing documentation designed to help developers integrate with and understand the Pet Shop API.

## 📚 Documentation Structure

### [Getting Started](docs/getting-started.md)
Quick start guide for developers new to the API. Learn how to set up your environment, make your first API call, and understand the basics.

### [Architecture](docs/architecture.md)
High-level overview of the API's architecture, design principles, and how the system is structured. Perfect for understanding the "why" behind design decisions.

### [API Reference](docs/api-reference.md)
Complete API reference with all endpoints, request/response formats, error codes, and examples. Your go-to resource for integration details.

### [Order Lifecycle](docs/order-lifecycle.md)
Deep dive into how orders work, state transitions, and business rules. Essential reading for implementing order-related features.

## 🎯 Quick Links

- **Base URL**: `http://localhost:7052/api/v1` (development)
- **Swagger UI**: `http://localhost:7052/swagger`

## 🚀 For First-Time Users

1. Start with [Getting Started](docs/getting-started.md) to set up your environment
2. Review [API Reference](docs/api-reference.md) for endpoint details
3. Read [Order Lifecycle](docs/order-lifecycle.md) to understand order states
4. Explore the [Architecture](docs/architecture.md) for deeper understanding
